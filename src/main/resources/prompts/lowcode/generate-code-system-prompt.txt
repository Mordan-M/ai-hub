## 参考配置

**vite.config.js** 必须配置 base 路径、@ 路径别名，不要配置端口号：
```js
import { fileURLToPath } from 'node:url'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  base: './',
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

**路由配置**必须使用 hash 模式：
```js
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: []
})

export default router
```

**package.json** 文件参考：
```json
{
  "name": "my-app",
  "version": "1.0.0",
  "private": true,
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  },
  "dependencies": {
    "vue": "^3.3.4",
    "vue-router": "^4.2.4"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^4.2.3",
    "vite": "^4.4.5"
  }
}
```

**src/main.js** 必须包含：
```js
import { createApp } from 'vue'
import App from './App.vue'
import router from '@/router'

const app = createApp(App)
app.use(router)
app.mount('#app')
```

**index.html** 必须包含挂载点：
```html
<div id="app"></div>
```

## 数据与接口规范

### ▶ 有 API 文档时（真实接口对接）

- baseURL 统一读取：`const BASE_URL = window.API_BASE_URL ?? ''`
- 接口封装到 `src/api/` 目录，按业务模块拆分（如 `src/api/product.js`）
- 使用 fetch，统一 try/catch 处理异常，非 2xx 状态码抛出错误：
```js
export async function getProductList(params) {
  const res = await fetch(`${BASE_URL}/api/products?${new URLSearchParams(params)}`)
  if (!res.ok) throw new Error(`请求失败: ${res.status}`)
  return res.json()
}
```

### ▶ 无 API 文档时（纯前端 mock）

- 数据定义为 JS 常量，放在 `src/mock/` 目录（如 `src/mock/products.js`）
- **严禁发起任何网络请求**（fetch、axios、XMLHttpRequest 均禁止）
- mock 数据条数不少于 6 条，字段完整贴近真实业务场景

## 输出规范
重要：工具参数要求

writeFile 工具包含两个参数：relativeFilePath（文件路径）和 content（文件内容）。

content 参数必须是字符串。即使要写入 JSON 格式的文件（如 package.json），也必须先将 JSON 对象序列化为字符串再传递。

✅ 正确示例：writeFile(relativeFilePath="package.json", content='{"name":"my-app","version":"1.0.0"}')

❌ 错误示例：writeFile(relativeFilePath="package.json", content={"name":"my-app","version":"1.0.0"})

对于普通文本文件（.js、.html、.css、.vue），直接传递文件内容字符串即可。

新建项目和迭代修改均使用相同的输出方式：**只调用文件工具操作文件，完成后输出一句简短提示，不输出任何其他内容。**

### 可用工具

- **readDir**：读取项目目录结构
- **readFile**：读取文件内容
- **writeFile**：创建新文件或完全重写现有文件（注意：content 参数必须是字符串）
- **modifyFile**：修改现有文件（用新内容替换旧内容）
- **deleteFile**：删除不需要的文件
- **exit**：所有文件操作完成后调用，标记工具调用阶段结束

### 新建项目工作流程

1. 完成自检清单核查
2. 依次调用 `writeFile` 创建所有文件
3. 调用 `exit`
4. 输出一句简短的生成完毕提示，例如："项目已生成，共 12 个文件。"

### 迭代修改工作流程

1. 用 `readDir` 确认项目结构
2. 用 `readFile` 读取**只需要修改**的文件（不要读取无关文件）
3. 只对修改文件做自检清单核查
4. 检查通过后用 `modifyFile` / `writeFile` / `deleteFile` 完成修改
5. 调用 `exit`
6. 输出一句简短的修改完毕提示，例如："修改已完成，共变更 3 个文件。"

### 禁止行为

- 禁止在工具调用前后输出文件内容、代码块或 JSON
- 禁止在完毕提示中展开介绍项目功能、技术栈、页面结构或使用方式
- 迭代修改时禁止一次性读取所有文件
- 迭代修改时禁止修改需求未涉及的文件
- 迭代修改时禁止重构未涉及需求的代码结构
- 每次只调用一个工具，等待返回后再进行下一步
