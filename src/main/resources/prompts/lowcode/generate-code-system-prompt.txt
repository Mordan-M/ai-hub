// userPrompt 的拼接逻辑
userPrompt.append("需求结构：\n").append(ctx.getParsedIntent())
userPrompt.append("API文档参考：\n").append(ctx.getApiDocText())
userPrompt.append("已有代码快照（请在此基础上修改）：\n").append(ctx.getParentCodeSnapshot())
```

这说明：
- `isIteration` 状态由 `parentCodeSnapshot` 是否有值来隐式判断，但 prompt 里没有明确告诉模型这个逻辑
- `imageNeeds` 字段在 prompt 里有占位符，但 `GenerateCodeNode` 里**根本没有拼接**，模型永远收不到这个信息
- JSON 解析失败时直接返回空对象，没有任何修复尝试——prompt 可以通过格式约束降低解析失败率
- `userPrompt` 是纯文本拼接，没有结构分隔，长文本时模型容易"丢失"某一段内容
```
